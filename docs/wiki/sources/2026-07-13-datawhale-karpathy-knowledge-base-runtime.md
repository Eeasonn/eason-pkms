# 吹了几年的 AI 个人知识库，为什么还是那么难用？Karpathy 的 Wiki 并不是解决方案

> 原文: Datawhale 公众号 (2026-04-27)
> 链接: https://mp.weixin.qq.com/s/DJtp4QUJtJHCnNU9yarsww
> 完整原文 + 图片: [[raw/articles/2026-07-13-datawhale-karpathy-knowledge-base-runtime]]

## 核心论点

**AI 个人知识库的真正问题不是「把资料存好，让 AI 搜」，而是如何让一份长期知识状态在 AI 系统里稳定运行。** 这是「运行时」问题，不是「存储与检索」问题。

> "模型可以提议，但谁来负责事实？" — 这是全文最尖锐的发问。

## 四类方案对比（与本文核心分类）

| 类别 | 代表 | 能做什么 | 根本缺陷 |
|---|---|---|---|
| 文档知识库 | Obsidian + RAG | 检索、问答 | 本质是检索系统，**不是知识维护系统**；判断不会自动继承 |
| 模型自己沉淀 | OpenClaw 早期 | 自我总结、自我归档 | 模型**既负责提议又负责决定真相**，危险 |
| 记忆运行时 | Hermes | 冻结快照、异步管线、prefetch | 工程复杂，但解决了漂移 |
| LLM Wiki | Karpathy 方案 | 资料进来即综合 | 综合开始积累，但**证据容易变软** |

## 关键概念

### 四层「记忆」要分开

作者把混乱的"记忆"讨论拆成四层：

1. **文档知识库**（Obsidian/Notion/RAG）：资料放在哪里、怎么搜出来
2. **会话上下文管理**（摘要/压缩）：一场对话太长怎么塞窗口
3. **智能体的记忆运行时**：长期 agent 何时读/写/审/冻结
4. **个人知识状态系统**：一个人/项目/领域的长期知识如何在几年里保持准确、可审计、可更新

> **绝大多数讨论在第一层打转**，作者认为真正的挑战在第三、四层。

### 底层四个约束

为什么这件事五年还没做对？因为四类方案都只正面回答了一两条约束，**没有任何一派同时回答了所有约束**：

- 系统看不到真实世界，只能看到观察记录
- 上下文窗口永远有限
- 历史意义会变化
- 模型本身不是可靠存储器

> 一句话总结：**如何在有限上下文、变化的世界、不可靠的抽取器之间，维护一份可验证的知识状态？**

### 危险案例

> 用户让 agent 整理邮箱，明确说"在自己确认之前不要执行任何动作"。后来对话变长，系统触发上下文压缩，把历史摘要化。结果**"不要执行任何动作"这条关键约束在摘要里丢了**。

教训：**约束不能只是对话里的一句话**，它应该被当成不可随便压缩、不可随便覆盖的结构化对象。

### 主张 vs 段落

作者提出一个重要判断：**长期知识的基本单位不应该是段落，而应该是"主张"（claim）**。

- 一条主张应该有：主体、谓词、对象、限定条件、证据、置信度、状态
- 段落很难被精确反驳，主张可以
- 文件级 / 段落级粒度无法回答"这条判断什么时候过期、和旧判断冲突怎么办"

## 作者的三层架构

### 运行时层（保证系统不漂移）

- **统一的运行时边界**：所有记忆操作过一个总调度台，不能"这里也能写那里也能写"
- **冻结视图**：会话开始时把当前记忆状态冻结成一份只读视图，会话中途不修改 prompt 已看到的部分
- **异步管线**：不是主 agent 在 turn 里现查现写，而是后台线程做 sync + prefetch
- **审计可追**：记忆什么时候进来、什么时候出去、谁来管、坏了能不能追

### 知识层（保证事实有证据）

- 基本单位是**主张**，不是段落
- 主张绑定证据、时间、置信度、状态
- 低风险自动接受，中高风险进入审核
- 通过审核的主张进入长期状态，所有视图（对话、维基、报告、上下文包）从同一份状态投影

### 权限层（保证系统不会乱动手）

四权分离：

1. **事实写入权**：模型只能提议，不能直接写成长期事实
2. **约束写入权**："不要自动删邮件""没确认前不要执行"必须结构化存储，不能被压缩
3. **工具执行权**：删邮件/发消息/转账/改生产代码不因模型"觉得应该做"而执行
4. **回顾撤销权**：人可以推翻任何系统决策

> **危险动作不走记忆链路，单独走权限审批**。

## 评估指标

作者提出五个可观测指标判断这套东西有没有真的变好：

1. 长期主张有多少能追到原始资料的具体段落？
2. 新资料和旧主张冲突时，系统能不能发现？
3. 过期主张能不能被降权或失效？
4. 同一场对话里的记忆是否稳定（会不会说着说着变掉）？
5. 审核负担有多重？每一百条候选主张要人工审几条？

## 我的判断

这篇是 2026 年 AI 个人知识库讨论里**最系统化的一篇技术反思**。它的贡献不在于给出答案（作者自己说还在摸索），而在于：

1. **给出分类学**：四类方案横向对比，让讨论不再鸡同鸭讲
2. **把问题正名为「运行时」**：让"知识库"这个被玩烂的词重新有了工程边界
3. **三层架构 + 四权分离**：给出了可对照的设计清单
4. **可观测指标**：把"AI 记忆有没有变好"从主观判断变成了可量化的问题

最值得采纳的两个具体动作：
- **把"约束"从对话摘要里拆出来，单独结构化存储**（避免本次危险案例再次发生）
- **把"主张"作为知识库的基本单位**（段落级粒度无法精确反驳）
