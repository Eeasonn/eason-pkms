# AI 记忆运行时 (AI Memory Runtime)

## 定义

AI 记忆运行时是**长期运行 agent 的记忆子系统**：负责"记忆什么时候进来、什么时候出去、谁来管、坏了能不能追"。它不是"agent 顺手用的功能"，而是 agent 的基础设施。

> 与文档知识库（RAG / Obsidian）的本质区别：RAG 是**检索系统**，运行时是**状态维护系统**——它维护一份"长期知识状态"，让今天得出的判断明天能自动继承。

## 三层架构

```
┌─────────────────────────────────────────┐
│  权限层（保证系统不会乱动手）             │
│  ─ 事实写入权 / 约束写入权 / 工具执行权   │
├─────────────────────────────────────────┤
│  知识层（保证事实有证据）                 │
│  ─ 主张（Claim）为基本单位                │
│  ─ 绑定证据、时间、置信度、状态           │
├─────────────────────────────────────────┤
│  运行时层（保证系统不漂移）               │
│  ─ 冻结视图 / 异步管线 / 审计追踪        │
└─────────────────────────────────────────┘
```

## 关键机制

### 1. 冻结视图（Session Snapshot）

会话开始时，把当前记忆状态冻结成一份**只读视图**给当前会话用。中途即使有新写入或审核通过的内容落盘，也不会修改当前会话已经看到的那份 prompt。

> **作用**：保证同一场对话里的记忆稳定——不会说着说着变掉。

### 2. 异步管线（Async Pipeline）

不是"主 agent 在 turn 里现查现写"，而是这一轮结束后：
1. 后台线程做 sync（落盘新内容）
2. 做 prefetch（把下一轮可能需要的记忆提前准备好）

> **作用**：降低主 agent 的认知负担，避免在 prompt 组装时临时拼装历史。

### 3. 主张作为知识单位（Claim-based Knowledge）

长期知识的基本单位不是文件/段落/Markdown 页面，而是**主张**：

```
{
  subject: "项目A",
  predicate: "uses",
  object: "事件驱动架构",
  qualifiers: { "since": "2024-03" },
  evidence: [doc_ref_1, doc_ref_2],
  confidence: 0.85,
  status: "active" | "pending_review" | "deprecated"
}
```

**为什么不用段落**：段落很难被精确反驳，主张可以。

### 4. 四权分离

| 权力 | 谁持有 | 边界 |
|---|---|---|
| **事实写入权** | 系统（自动接受低风险）/ 人工（中高风险） | 模型只能提议，不能直接写成长期事实 |
| **约束写入权** | 人工 | "不要自动删邮件""没确认前不要执行"必须结构化存储，不可被压缩覆盖 |
| **工具执行权** | 人工 | 删邮件/发消息/转账/改生产代码不因模型"觉得应该做"而执行 |
| **回顾撤销权** | 人工 | 人可以推翻任何系统决策 |

> **核心原则**：危险动作不走记忆链路，单独走权限审批。

## 评估指标

判断一套 AI 记忆系统有没有真的变好，看五个指标：

1. **可追溯性**：长期主张中有多少能追到原始资料的具体段落？
2. **冲突发现**：新资料和旧主张冲突时，系统能不能发现？
3. **过期降权**：过期主张能不能被降权或失效？
4. **对话稳定性**：同一场对话里的记忆是否稳定（会不会说着说着变掉）？
5. **审核负担**：每一百条候选主张要人工审几条？

> 如果每一百条候选主张要人工审九十条，用户迟早会放弃。

## 与现有 AI 系统的关系

| 系统 | 层级 | 缺失 |
|---|---|---|
| RAG / Obsidian | 文档知识库 | 没有运行时 |
| OpenClaw（早期模式） | 模型自我沉淀 | 模型既提议又决定真相，危险 |
| **Hermes** | 真正运行时 | 已实现冻结视图 + 异步管线 |
| Karpathy LLM Wiki | 文档综合 | 证据容易变软，没有权限层 |

## 关键洞察

> **"模型可以提议，但谁来负责事实？"**

这是 AI 记忆系统设计的核心问题。三层架构、四权分离、五个指标，都是这个问题在不同维度的展开。
